# Release process — fissible projects

All fissible projects follow [Semantic Versioning](https://semver.org) driven by
[Conventional Commits](https://www.conventionalcommits.org).

---

## Version bump rules

| Commits since last release contain… | Bump |
|---|---|
| `feat!:` or `BREAKING CHANGE:` in footer | **major** (`1.0.0 → 2.0.0`) |
| `feat:` | **minor** (`1.0.0 → 1.1.0`) |
| `fix:` only | **patch** (`1.0.0 → 1.0.1`) |
| `chore:` / `docs:` / `test:` only | no release needed |

When in doubt, err toward a minor bump over a patch.

---

## Prerequisites

Every fissible repo must have:

| File | Purpose |
|---|---|
| `VERSION` | Single source of truth for the current version (one line, no `v` prefix) |
| `CHANGELOG.md` | Keep a Changelog format, generated by git-cliff |
| `.cliff.toml` | git-cliff config (copy from `fissible/.github`) |

Install git-cliff once:
```bash
brew install git-cliff
```

---

## Cutting a release

Use `release.sh` (copy from `fissible/.github` into your repo's root, or run it directly):

```bash
bash release.sh          # auto-detects bump type
bash release.sh minor    # or specify: patch | minor | major
```

The script will:
1. Show commits since the last tag
2. Suggest a bump type (or accept your override)
3. Update `VERSION`
4. Regenerate `CHANGELOG.md` via git-cliff
5. Commit `chore: release vX.Y.Z`
6. Create annotated tag `vX.Y.Z`
7. Push branch and tags (with confirmation)

---

## What the release script does NOT do

- Update a Homebrew formula — do that manually in `fissible/homebrew-tap` after pushing the tag
- Bump versions in `composer.json`, `package.json`, etc. — update those before running the script

---

## Manual steps (if not using release.sh)

```bash
# 1. Determine bump — scan commits since last tag
git log $(git describe --tags --abbrev=0)..HEAD --oneline --no-merges

# 2. Update VERSION
echo "1.2.0" > VERSION

# 3. Regenerate CHANGELOG
git-cliff --config .cliff.toml --tag v1.2.0 --output CHANGELOG.md

# 4. Commit and tag
git add VERSION CHANGELOG.md
git commit -m "chore: release v1.2.0"
git tag -a v1.2.0 -m "v1.2.0"

# 5. Push
git push && git push --tags
```

## After releasing

- [ ] Verify the GitHub Release was created automatically (if the release workflow is enabled)
- [ ] Update `fissible/homebrew-tap` formula if the project is distributed via Homebrew
- [ ] Update dependent projects if this was a breaking change
